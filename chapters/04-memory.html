4. 持久化记忆

🧠
MiMo Code有超强记忆力！它能记住你的项目、你的偏好，甚至你上次聊到哪了！

4.1 记忆系统的神奇之处

普通AI助手每次对话都像"失忆"了一样，但MiMo Code不一样——它能记住你！

我上次让你帮我写那个登录功能...
我记得！你当时说要用JWT认证，我已经写了一半了...
对对对！继续吧！

记忆 = 跨会话的超能力

关掉终端再打开，MiMo Code还记得你之前在做什么！

4.2 记忆的四个仓库

MiMo Code把记忆分成了四个部分，就像一个井井有条的图书馆：

记忆类型 存什么 比喻
项目记忆 项目规则、架构决策 项目说明书
会话检查点 当前工作状态 书签
临时笔记 随手记的想法 便利贴
任务进度 每个任务的进展 待办清单

4.3 记忆是怎么工作的？

当你开始新会话时，MiMo Code会自动：

  1. 加载上次的"书签"（检查点）
  2. 翻阅"项目说明书"（项目记忆）
  3. 查看"待办清单"（任务进度）
  4. 继续你之前的工作
✨
整个过程自动完成，你什么都不用管！

4.4 告诉MiMo Code你的规则

教它你的规矩

告诉MiMo Code你的项目规范，它会记住并在以后遵守！

我们项目用TypeScript，所有函数都要写注释
明白了！我会记住这个规则，以后写代码都会加注释。
变量名要用驼峰命名法
收到！以后变量名都用驼峰命名。
💡 这些规则会被保存到 MEMORY.md 文件中，你可以随时查看和修改。

4.5 跨会话继续工作

这是记忆系统最实用的功能：

# 今天下午
> 帮我实现用户登录功能
# MiMo Code开始工作...写了一半

# 关掉终端，去喝杯咖啡 ☕

# 晚上回来
> mimo
> 继续上次的工作
# MiMo Code：我记得！我们登录功能写到...
☕
去喝杯咖啡、吃个饭、睡一觉，回来它还记得！

4.6 最佳实践

✅ 这样做

主动告诉项目规范

让系统自动管理检查点

定期查看 MEMORY.md

❌ 别这样做

每次都重新介绍项目

手动修改检查点文件

忽视记忆系统的提示
