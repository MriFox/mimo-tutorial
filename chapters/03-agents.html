3. 三种代理模式

🎭
MiMo Code有三个"分身"，每个都有不同的超能力！学会切换它们，你就能应对各种场景！

3.1 认识三个小伙伴

想象一下，你有三个不同性格的助手：

Build说："交给我！我来帮你写代码、改bug！"
Plan说："先别急，让我帮你分析分析..."
Compose说："我来帮你安排整个开发流程！"

3.2 Build - 全能选手

Build：你的主力助手

能读能写能跑命令，开发工作全靠它！

Build是默认模式，它能做任何事：

  • 读写代码文件
  • 运行程序和命令
  • 管理Git版本控制
  • 创建子代理帮忙
🔧
就像一个全能的工具人，什么活都能干！

什么时候用Build？

帮我写一个函数
这个bug怎么修？
运行一下测试

3.3 Plan - 分析师

Plan：你的智囊团

只看不动手，帮你分析问题、设计方案！

Plan是个"只读"模式，它只能看不能改：

  • 只能读取文件
  • 不能修改任何内容
  • 专注于分析和建议
🧠
就像军师一样，先想清楚再动手！

什么时候用Plan？

这个项目用了什么技术？
帮我分析一下这段代码
这个功能应该怎么设计？

3.4 Compose - 总导演

Compose：你的项目经理

安排整个开发流程，从规划到测试一条龙！

Compose是"编排"模式，它能安排复杂的工作流：

  • 规格驱动开发
  • 技能驱动工作流
  • 全生命周期管理
🎬
就像电影导演，统筹安排所有环节！

什么时候用Compose？

帮我开发一个完整功能
按照规范写代码
需要代码审查和测试

3.5 如何切换？

一个键搞定！

按 Tab 键就能在三个模式之间切换！

当前模式: Build
按 Tab →
当前模式: Plan
按 Tab →
当前模式: Compose
按 Tab →
当前模式: Build（循环了）

3.6 实战演练

动手试试看！

跟着下面的步骤操作，体验三种模式的区别

演练1：Plan模式分析代码

# 按 Tab 切换到 Plan 模式
# 然后输入：
> 分析这个项目的目录结构，告诉我它使用了什么技术栈
> 这段代码有什么潜在问题？
> 设计一个用户登录系统的架构

演练2：Build模式写代码

# 按 Tab 切换到 Build 模式
# 然后输入：
> 帮我写一个Python函数，计算斐波那契数列
> 在这个文件中添加错误处理
> 运行一下测试看看有没有问题

演练3：Compose模式开发功能

# 按 Tab 切换到 Compose 模式
# 然后输入：
> 帮我开发一个简单的计算器功能
> 需求：支持加减乘除，有输入验证，包含单元测试

3.7 选择建议

✅ 新手推荐

先用Plan模式了解项目

再用Build模式动手修改

复杂项目用Compose

❌ 避免这样做

上来就用Build乱改

用Plan模式修改文件

简单任务用Compose（杀鸡用牛刀）
